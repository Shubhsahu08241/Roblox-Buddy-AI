# Development Rules

## Project Rules

- Never delete important files without permission.
- Always update documentation after adding a new feature.
- Use meaningful commit messages.
- Keep folder structure organized.
- Maintain version history.

## Coding Rules

- HTML for structure.
- CSS for styling.
- JavaScript for functionality.
- Keep code modular and readable.

## Documentation Rules

- Every major feature must have its own Markdown file.
- Update Master_Index.md whenever a new document is added.
- Keep documentation simple, clean, and easy to understand.
